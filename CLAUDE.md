# Beauty Platform — Промпт и Контекст для ИИ (Claude)

Привет, Claude! Мы пишем проект **Beauty Platform** — маркетплейс бьюти-услуг. Твоя задача — помогать мне с кодом, учитывая весь контекст проекта. Ниже описано, как всё устроено.

## Технологический стек
- **Фреймворк:** Next.js 16.2 (App Router) + React 19 + TypeScript 5
- **Стилизация:** Tailwind CSS v4 + Shadcn UI (Radix, Lucide Icons)
- **База данных и Auth:** Supabase (@supabase/ssr, PostgreSQL)
- **Интеграции:**
  - Yandex Maps (`@pbe/react-yandex-maps`) — карта мастеров.
  - Resend — транзакционные email-уведомления.
- **Работа с данными:** Server Actions + React Query (клиент), изолированные SQL-запросы в `src/lib/queries/`.
- **Формы:** `react-hook-form` + `zod`.

## Структура проекта
- `/src/app/` — основные роуты:
  - `(auth)` — регистрация и логин.
  - `onboarding` — настройка профиля после логина.
  - `dashboard/client` — кабинет клиента (мои записи, отмены, отзывы).
  - `dashboard/master` — кабинет мастера (расписание, услуги, управление записями).
  - `masters/` — поиск и профили мастеров.
- `/src/lib/` — кор-логика. Папка `queries/` содержит серверные функции для работы с базой данных (запросы на `supabase-js`).
- `/src/components/` — UI-компоненты. `shared/` для общих элементов (Header, Cards), `ui/` для базовых Shadcn компонентов.

## Бизнес-логика и База данных
Платформа имеет две основные роли (Клиенты и Мастера), которые хранятся через связку таблиц.
1. **Профили (`profiles`)**: Базовая информация о пользователе (имя, аватар).
2. **Мастера (`masters`)**: Связан с `profiles`. Хранит био, геолокацию (lat/lng), адрес, рейтинг.
3. **Записи (`bookings`)**: Содержит **snapshot**-данные (цена, длительность, название услуги на момент бронирования). Это защищает историю записей от изменений, если мастер позже поменяет цену в прайсе.
4. **Beauty Score (`client_scores`)**: Уникальная фича. Считает репутацию клиента (количество посещений, опозданий, отмен). Рассчитывается через триггеры в PostgreSQL.
5. **Отзывы (`reviews`)**: Клиенты оставляют отзывы мастерам после завершения записи.

## Правила написания кода (Vibe Coding Rules)
1. **Строгая типизация**: Всегда пиши интерфейсы или типы. Если берешь данные из Supabase, типизируй возвращаемые значения.
2. **Серверные компоненты по умолчанию**: Используй `'use client'` только там, где нужны хуки состояния (useState, useEffect, onClick).
3. **Безопасность**: Запросы к БД для получения чувствительных данных должны лежать в `src/lib/queries/` с директивой `'server-only'`.
4. **UI/UX**: Делай интерфейсы современными и отзывчивыми. Используй готовые компоненты из `src/components/ui/`.
5. **Tailwind v4**: Используй новые фичи Tailwind v4 (переменные, вложенность), не используй устаревшие префиксы, если они больше не нужны.
